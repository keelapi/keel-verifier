# Public Docs Policy

Public docs should describe stable behavior, supported surfaces, limitations,
evidence formats, and interoperability expectations.

Internal research notes, verification diaries, source-vetting analysis, roadmap
speculation, legal/compliance workpapers, and raw security scratchpads belong
in ignored local-only folders such as `_internal-local/`, `research-local/`,
or `compliance-research-local/`.

Public limitations should remain clear and accurate, but should be written as
product or protocol boundaries rather than internal research commentary.
